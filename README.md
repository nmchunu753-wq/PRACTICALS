# Google BigQuery Practical Exercises - Retail Sales Analysis

## 📌 Overview

This repository contains practical SQL exercises and solutions for **Google BigQuery**, focused on retail sales data analysis. The exercises are designed to help data analysts, data scientists, and SQL learners master essential BigQuery concepts through hands-on practice with real-world retail scenarios.

## 🎯 Purpose

This repository serves as a comprehensive practical guide for:
- Learning Google BigQuery SQL syntax and best practices
- Mastering data manipulation techniques for retail analytics
- Building a strong foundation in data querying and aggregation
- Preparing for data analysis roles and SQL interviews

## 📊 Dataset

All exercises are based on a **retail sales dataset** (`sales-projects-494801.Sales.Retailsales`) containing:
- Transaction details
- Customer demographics (Age, Gender)
- Product categories
- Sales amounts, quantities, and pricing
- Temporal data (transaction dates)

## 📚 Exercise Topics Covered

| # | Topic | Description |
|---|-------|-------------|
| 1 | **WHERE Clause** | Filtering transactions by date/year |
| 2 | **Filtering + Conditions** | Using subqueries to filter above average values |
| 3 | **GROUP BY** | Aggregating total quantity by product category |
| 4 | **Aggregate Functions** | Calculating sums, averages, and counts |
| 5 | **Date Functions** | Extracting year, month from date columns |
| 6 | **CASE Statements** | Creating age group classifications |
| 7 | **Conditional Aggregation** | Counting high-value transactions by gender |
| 8 | **HAVING Clause** | Filtering aggregated results (total revenue > 5000) |
| 9 | **Calculated Fields** | Creating unit cost categories (Cheap/Moderate/Expensive) |
| 10 | **WHERE + CASE** | Combining filters with conditional logic |

## 🛠️ Technologies Used

- **Google BigQuery** - Serverless, highly scalable data warehouse
- **SQL** - Standard SQL syntax with BigQuery extensions
- **Google Cloud Platform (GCP)** - Data storage and processing
